# Multi Order Client Demo Script

## Demo Goal

Show that Multi Order reduces repetitive ordering while preserving billing, kitchen, delivery, and admin control.

## Script

1. Open Family module and show the new `Multi Order` hub card.
2. Create a Lunch multi order for one linked student across several weekdays.
3. Show the review step with the generated date list.
4. Submit the group and confirm the new grouped entry appears in the Family multi-order page.
5. Open Family billing and show the grouped billing row.
6. Upload grouped payment proof from Family billing.
7. Open Admin billing and show the grouped billing row pending verification.
8. Verify the grouped billing from Admin.
9. Generate the grouped receipt.
10. Open Admin Multi Orders and show the occurrence breakdown.
11. Submit a post-start change request from Family or Student.
12. Resolve that request from Admin Multi Orders.
13. Show that kitchen and delivery still operate on normal occurrence orders.

## Key Messages

- One grouped action creates many future orders.
- Billing stays clean through one grouped billing row.
- Operational teams do not need a separate kitchen or delivery process.
- Admin keeps control over changes after the plan has started.
